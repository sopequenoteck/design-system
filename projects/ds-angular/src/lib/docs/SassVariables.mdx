# Variables Sass

Variables SCSS disponibles pour personnaliser le design system au niveau du build.

---

## Architecture des tokens

Le design system utilise une architecture à 3 couches :

```
_primitives.scss    →  Variables de base (couleurs, tailles)
      ↓
_semantic.scss      →  Variables sémantiques (composants)
      ↓
_tokens.scss        →  CSS custom properties (:root)
```

## Utilisation

### Dans vos fichiers SCSS

```scss
@use '@kksdev/ds-angular/styles/tokens/primitives' as primitives;
@use '@kksdev/ds-angular/styles/tokens/semantic' as semantic;

.my-component {
  background: primitives.$gray-100;
  padding: primitives.$space-4;
  border-radius: semantic.$card-radius;
}
```

### Surcharge des variables

Pour personnaliser les valeurs par défaut :

```scss
// Avant l'import des styles
$primary: #your-brand-color;
$radius-2: 4px; // Coins moins arrondis

@use '@kksdev/ds-angular/styles' with (
  $primary: $primary,
  $radius-2: $radius-2
);
```

## Bonnes pratiques

1. **Utilisez les variables sémantiques** plutôt que les primitives dans vos composants
2. **Ne modifiez pas** les variables primitives directement dans node_modules
3. **Créez vos propres variables** qui référencent celles du DS pour faciliter la maintenance
4. **Documentez** vos surcharges dans un fichier dédié

```scss
// _my-variables.scss
$my-primary: #your-color;
$my-card-radius: 4px;

// Utilisez ensuite ces variables dans vos composants
```
